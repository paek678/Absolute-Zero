### Q18. 스마트폰 회복량 3→5→7° vs 1회용 모순 `아이템` `시스템`

스마트폰은 **RandomConsumable(1회용)**으로 분류되어 있는데, 회복량이 **3→5→7°**로 3단계입니다.

1회용이면 항상 3°만 회복하고 사라져야 하고, 3단계면 3회 사용 가능해야 합니다.

> 현재 코드: `HealPerUse = {3, 5, 7}` 배열로 다회 사용 지원하지만, RandomConsumable 타입이라 1회 후 삭제

- [ ] 1회용, 항상 3° 고정
- [ ] 3회 사용 가능 (3→5→7°)
- [ ] 기타: ___

### Q19. 안아줘요 티셔츠 — 내 온도가 더 높을 때 역효과 `아이템` `시스템`

안아줘요 티셔츠 효과: "상대 온도를 내 온도와 같게 만듦"

내 온도(25°)가 상대(15°)보다 높으면 → 상대 온도를 25°로 **올려주는** 결과 (상대를 도와줌).

> 현재 코드: `diff = 상대 - 나`. diff < 0이면 `ApplyHeal(상대, -diff)` → 상대 회복

- [ ] 의도된 리스크 (사용자 판단 필요)
- [ ] 내 온도 ≥ 상대면 효과 없음
- [ ] 항상 상대 온도를 내림 (절대값 차이)

## 해결된 질문 — 8개

### ✅ Q4. 삼계탕 "+3, -7" 효과 방향

즉시 상대 온도 +3° (올림/회복) → 다음 턴 상대 온도 -7° (내림/공격). 둘 다 상대에게 적용.

### ✅ Q5. 탄산음료 "-5, +15" 효과 방향

즉시 내 온도 -5° (자해 비용) → 다음 턴 내 온도 +15° (이득). 둘 다 자신에게 적용.

### ✅ Q11. 미니게임 판정 권한

클라이언트에서 판정 → 성공/실패 결과만 ServerRpc로 전송.
서버 타이머 만료 시 → 강제 실패 처리 (서버가 마스터).

### ✅ Q12. 미니게임 실패 시 아이템 소모

소모 안 됨. 실패해도 아이템 유지, 다시 선택하여 재도전 가능.

### ✅ Q13. 공격턴 아이템 사용 연출

전용 애니메이션/이펙트 예정. 아트 확정 후 결정.

### ✅ Q14. 선풍기 월드 표시

StayItem 위치에 오브젝트 스폰만. 연출은 추후 추가.

### ✅ Q15. 상대방 아이템 보유 목록

EnemyItem 위치에 서버 데이터 기반 표시. 서버가 양쪽 인벤토리 관리 → 클라이언트는 표시만.
소모 시 시각적 제거 + 오브젝트 삭제, 재획득 시 다시 표시.

### ✅ Q17. 환경 시스템
